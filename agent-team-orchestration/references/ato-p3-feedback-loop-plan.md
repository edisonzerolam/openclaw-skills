# P3 反馈闭环优化方案

**版本**: v1.0  
**创建时间**: 2026-05-24  
**优先级**: P3

---

## 1. 背景

### 1.1 问题
当前系统缺乏用户反馈机制：
- 无法知道推荐是否被用户采纳
- 无法根据实际使用情况优化匹配算法
- 专家知识文件质量无法量化评估

### 1.2 目标
建立完整的反馈闭环，实现持续优化。

---

## 2. 反馈收集机制

### 2.1 显式反馈
- 点赞/点踩推荐结果
- 选择"更符合需求"的专家
- 文本反馈（可选）

### 2.2 隐式反馈
- 点击率（用户是否查看专家详情）
- 停留时间（查看详情的时长）
- 最终选择（用户实际调用的专家）

### 2.3 反馈数据结构
```json
{
  "query": "用户原始查询",
  "recommended": ["expert1", "expert2", "expert3"],
  "selected": "expert1",
  "feedback": {
    "explicit": "thumbs_up",
    "implicit": {
      "click_rank": 1,
      "dwell_time": 45
    }
  },
  "timestamp": "2026-05-24T10:30:00Z"
}
```

---

## 3. 反馈处理流程

```
用户交互 → 反馈收集 → 数据存储 → 分析聚合 → 模型更新 → 效果验证
```

### 3.1 数据存储
- 本地 JSONL 文件（轻量级）
- 每日轮转，保留 90 天

### 3.2 分析聚合
- 每日统计各专家的点击率、采纳率
- 识别低质量专家文件（点击率 < 30%）
- 识别高频未匹配查询（改进索引）

---

## 4. 自动优化触发

### 4.1 触发条件
| 条件 | 阈值 | 动作 |
|------|------|------|
| 反馈数据量 | >= 100条 | 启动分析 |
| 专家点击率 | < 30% | 标记待优化 |
| 查询未匹配率 | > 20% | 触发索引更新 |
| 整体采纳率 | < 60% | 触发模型重训练 |

### 4.2 优化动作
1. **索引更新**: 添加新关键词，调整权重
2. **专家文件更新**: 标记低质量文件，提示人工审核
3. **模型微调**: 基于反馈数据调整匹配算法参数

---

## 5. 实施步骤

### Step 1: 反馈收集模块（2天）
- 创建 feedback_collector.py
- 实现显式和隐式反馈收集

### Step 2: 数据存储（1天）
- 设计反馈数据结构
- 实现 JSONL 存储和轮转

### Step 3: 分析模块（2天）
- 创建 feedback_analyzer.py
- 实现统计分析和可视化

### Step 4: 自动优化（2天）
- 实现触发条件检测
- 集成索引更新和模型微调

### Step 5: 监控面板（2天）
- 创建反馈监控仪表盘
- 展示关键指标趋势

---

## 6. 验证标准

- 反馈收集率 > 50%（用户交互中至少一半有反馈）
- 数据分析延迟 < 1小时
- 自动优化触发准确率 > 80%
- 整体采纳率提升 > 10%
