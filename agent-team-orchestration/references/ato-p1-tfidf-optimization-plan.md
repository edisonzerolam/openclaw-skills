## 1. 当前问题诊断

### 1.1 问题描述
agent-team-orchestration 技能使用 TF-IDF 进行中文关键词匹配，当前存在以下问题：

1. **置信度阈值固定**: 当前使用固定阈值（0.6），未考虑不同查询类型的差异
2. **缺乏 n-gram 支持**: 仅支持单字/单词匹配，无法识别多词组合（如"多智能体编排"）
3. **未处理同义词**: "编排"和"协调"被视为不同词汇
4. **缺乏反馈学习**: 无法根据用户实际选择优化匹配权重

### 1.2 影响范围
- 专家推荐准确率：约 65%（目标 85%+）
- 冷数据区加载命中率：约 40%（目标 70%+）
- 用户需要多次尝试才能找到合适的专家

## 3. 具体实施步骤

### Step 1: 置信度阈值动态调优（2天）

**任务**:
1. 分析历史查询日志，统计不同查询类型的最佳阈值分布
2. 实现自适应阈值算法：
   ```python
   def adaptive_threshold(query_type, query_length, historical_accuracy):
       base = 0.6
       if query_type == "technical":
           base += 0.1
       if query_length < 5:
           base -= 0.1
       return min(0.85, max(0.4, base))
   ```
3. A/B 测试验证效果

**交付物**:
- `scripts/adaptive_threshold.py`
- 阈值调优报告

### Step 2: n-gram 增强（3天）

**任务**:
1. 实现 Bigram/Trigram 提取：
   ```python
   def extract_ngrams(text, n=2):
       tokens = jieba.lcut(text)
       return [''.join(tokens[i:i+n]) for i in range(len(tokens)-n+1)]
   ```
2. 更新索引构建流程，包含 n-gram 特征
3. 调整 TF-IDF 权重计算，融合 unigram 和 n-gram 得分

**交付物**:
- `scripts/ngram_indexer.py`
- 更新后的 `keyword-index.md` v1.3

### Step 3: 同义词扩展（2天）

**任务**:
1. 构建领域同义词表：
   ```yaml
   synonyms:
     编排: [协调, 调度, 管理, orchestration]
     智能体: [agent, 代理, 机器人]
     工作流: [workflow, 流程, 流水线]
   ```
2. 查询时自动扩展同义词
3. 索引构建时合并同义词权重

**交付物**:
- `references/synonyms.yaml`
- 同义词扩展模块

### Step 4: 集成与验证（1天）

**任务**:
1. 整合上述优化到 `team-brain.py`
2. 回归测试确保现有功能正常
3. 性能基准测试

## 5. 风险与回滚方案

| 风险 | 概率 | 影响 | 回滚方案 |

| n-gram 导致索引膨胀 | 中 | 高 | 限制 n-gram 长度，仅保留高频组合 |
