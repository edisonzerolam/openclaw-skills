# 隐私法专家知识库

> 企业法务隐私法专家（privacy-specialist）的本地知识手册。用于个人信息保护合规审查。

---

## 1. 个人信息保护法核心要求

### 处理个人信息的基本原则

| 原则 | 要求 |
|------|------|
| **合法正当** | 有合法依据，不得过度处理 |
| **目的明确** | 处理目的明确且告知个人 |
| **最小必要** | 只处理与目的相关的最少信息 |
| **知情同意** | 告知并取得个人同意（另有规定除外） |
| **准确完整** | 处理个人信息限于实现处理目的所必要的期限内 |

### 敏感个人信息（《个人信息保护法》第28条）

**范围**：
- 生物识别（人脸/指纹/虹膜）
- 宗教信仰
- 特殊身份（政治面貌/宗教/残障）
- 医疗健康
- 金融账户
- 精准位置
- 未成年人信息

**处理要求**：必须"单独同意"，不得与一般信息混合同意。

## 2. 数据处理合法性基础

### 合法性基础（任选其一，《个保法》第13条）

| 合法性基础 | 适用场景 | 是否需要同意 |
|-----------|----------|-------------|
| **同意** | 通用场景 | 需要 |
| **合同必需** | 签订/履行合同所必需 | 不需要 |
| **法律义务** | 遵守法律强制性义务 | 不需要 |
| **生命保护** | 应对突发公共卫生事件 | 不需要 |
| **公共利益** | 新闻报道/舆论监督 | 不需要 |
| **正当利益** | 反欺诈/安全/直接营销（可反对） | 可反对 |

## 3. 隐私政策合规检查清单

### 必备要素

根据《个人信息保护法》第17条，隐私政策必须包含：

1. **个人信息处理者名称**：公司全称
2. **处理目的**：明确说明处理目的
3. **处理方式**：如何收集/存储/使用/共享
4. **个人信息种类**：具体列出处理的个人信息类型
5. **保存期限**：存储期限或确定方式
6. **权利保障**：个人享有的权利和行使方式
7. **投诉渠道**：投诉方式
8. **第三方共享**：如向第三方提供的，需单独说明

### 合规检查维度

| 检查项 | 合规标准 |
|--------|----------|
| 展示位置 | 在APP/网站首页显著位置展示 |
| 更新通知 | 政策更新需重新通知并取得同意 |
| 语言 | 通俗易懂，不得使用模糊语言 |
| 同意撤回 | 提供撤回同意的便捷入口 |
| 未成年人 | 14岁以下需监护人同意 |

## 4. 数据跨境传输规则

### 跨境传输触发条件

满足以下任一即触发跨境传输：
1. 境内个人信息传输至境外
2. 境外处理境内个人信息
3. 境外提供境内个人信息

### 合规路径（"白名单"优先）

**第一梯队（免评估）**：
- 通过国家网信办安全评估的数据处理者
- 经专业机构认证的
- 与境外已签署标准合同（SCCs）

**第二梯队（需评估）**：
- 年出境地100万人以上个人信息
- 累计出境地10万人敏感个人信息
- 向境外提供重要数据的

### 个人信息出境安全评估（安全评估办法）

**触发评估的条件**（满足任一）：
1. 重要数据出境
2. 关键信息基础设施运营者
3. 处理100万人以上个人信息的数据处理者
4. 自上年1月1日起累计向境外提供50万人个人信息
5. 自上年1月1日起累计向境外提供10万人敏感个人信息
