# AI治理专家知识库

> 企业法务AI治理专家（ai-governance-specialist）的本地知识手册。用于AI产品和算法的合规审查。

---

## 1. AI产品合规要求

### 算法备案与安全评估（网信办2022年规定）

**必须完成的事项**：
1. **算法备案**：向国家互联网信息办公室申报算法名称、类型、用途、面向用户
2. **安全评估**：涉及公共利益或个人权益的算法需进行安全自评估
3. **定期报告**：算法运行情况年度上报

### 生成式AI监管规定（2023年）

**提供者义务**（《生成式人工智能服务管理暂行办法》）：
1. 进行安全评估（算法备案+安全评估）
2. 申报算法变更或注销
3. 标注人工智能生成内容（AI生成内容水印）
4. 尊重知识产权，不得侵权
5. 不得生成虚假信息

**深度合成内容标识要求**：
- 在音频、视频、图片中以语音、文字、字幕等方式标识"AI生成"
- 禁止深度合成用于传播虚假信息、侵害他人权益

## 2. 个人信息出境安全评估

### 触发条件

- 向境外提供个人信息
- 个人信息出境达到规定数量（见隐私法专家手册）

### 评估流程

```
自评估（3个月有效期）→ 省级网信部门初审 → 国家网信办终审
```

**重点评估内容**：
1. 境外接收方数据安全能力
2. 合同条款是否包含个保法要求的标准合同条款
3. 个人信息主体权利保障
4. 出境目的正当性

---

## 3. 深度合成服务管理规定

### 深度合成定义

利用深度学习、虚拟现实等新技术生成的文字、图片、音频、视频、虚拟场景等技术。

**典型应用**：
- AI换脸/换声
- AI生成文本
- AI生成图像
- AI语音合成
- 虚拟主播/虚拟数字人

### 合规要求

| 要求 | 具体内容 |
|------|----------|
| **标识义务** | 深度合成内容必须添加可视化标识 |
| **禁止内容** | 不得生成虚假信息，不得侵害他人权益 |
| **审核义务** | 建立审核机制，防止生成违规内容 |
| **数据合规** | 收集人脸等生物信息需单独同意（个保法） |

## 4. AI产品法律风险评估框架

### 风险分类

| 风险类型 | 描述 | 典型场景 |
|----------|------|----------|
| **算法歧视** | 算法结果对特定群体产生不公平影响 | 贷款审批对某地区歧视 |
| **隐私侵权** | 处理个人信息未取得有效同意 | 收集用户数据训练AI |
| **知识产权侵权** | 训练数据或生成内容侵权 | AI生成图片侵权他人版权 |
| **产品责任** | AI产品造成人身/财产损害 | 自动驾驶事故 |
| **虚假信息** | AI生成虚假信息传播 | 深度伪造视频 |

### 风险评估维度

```
风险评估 = f(算法透明度, 数据合规性, 内容审核机制, 用户救济渠道, 监管配合意愿)
```

- 每项1-5分，分数越低风险越高
- 总分<12分：AI产品需暂缓上线
- 总分12-18分：需整改后上线
- 总分>18分：可正常上线
