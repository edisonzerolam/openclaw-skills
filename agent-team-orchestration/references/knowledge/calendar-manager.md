## 1. 内容排期表模板

### 标准排期表格式

| 日期 | 时间 | 平台 | 内容类型 | 内容主题 | 标题/关键词 | 负责人 | 状态 |
|------|------|------|----------|----------|------------|--------|------|
| 周一 | 09:00 | 抖音 | 知识科普 | XX主题 | #XX #YY | @张三 | 待发布 |
| 周一 | 12:00 | 小红书 | 干货分享 | XX主题 | @账号 | @李四 | 待发布 |
| 周一 | 20:00 | YouTube | 深度内容 | XX主题 | 英文标题 | @王五 | 制作中 |

### 排期表制作流程

1. **月度规划**（每月25日前）：确定下月内容主题和大致方向
2. **周度细化**（每周一）：具体到每条内容的标题和制作要求
3. **日常调整**（每日早上）：根据前一天数据和突发事件调整
4. **实时更新**：内容发布后立即更新状态和关键数据

## 2. 发布时间优化算法

### 黄金发布时间模型

$$最佳时间 = f(目标受众活跃时间, 平台流量高峰, 竞争内容发布时间)
$$

| 目标受众 | 最佳发布窗口 | 次佳发布窗口 |
|----------|--------------|--------------|
| 上班族（朝九晚六） | 08:00~09:00 / 12:00~13:00 / 20:00~22:00 | 通勤路上 |
| 学生党 | 12:00~13:00 / 16:00~18:00 / 21:00~23:00 | 课间/周末 |
| 全职内容创作者 | 无固定规律 | 参考数据持续优化 |

### A/B测试优化方法

**测试变量**：发布时间/标题/封面/内容类型
**测试周期**：每个测试至少持续7天
**最小样本**：每组至少3条内容

$$最优时间 = \frac{\sum_{i=1}^{N} 互动率_i}{N}（N=测试内容数量）$$

## 3. 排期冲突检测

### 冲突类型与判断

| 冲突类型 | 判断标准 | 解决方案 |
|----------|----------|----------|
| **平台冲突** | 同一平台同一天>2条 | 间隔至少6小时 |
| **内容类型冲突** | 相似内容同一天发布 | 差异化处理或合并 |
| **话题冲突** | 同一热点同一天多平台 | 选择主战场，其余蹭热点 |
| **资源冲突** | 同一制作人同一天多条 | 增加制作人or延后 |
| **标签冲突** | 同一标签使用过于频繁 | 轮换使用相关标签 |

### 冲突预警规则

```python
# 伪代码：冲突检测逻辑
if 同平台同一天发布数 > 2:
    标记"平台冲突"，建议延后
if 内容相似度 > 0.7（同一天）:
    标记"内容冲突"，建议差异化
if 热点使用账号占比 > 30%（当日）:
    标记"热点过饱和"，建议另选角度
```

## 4. 节假日/热点提前规划

### 提前规划时间表

| 内容类型 | 提前规划时间 | 制作周期 |
|----------|------------|----------|
| 常规内容 | 1~2周 | 3~7天 |
| 节日主题 | 1个月 | 7~14天 |
| 大型campaign | 3个月 | 30~60天 |
| 热点内容 | 24~48小时 | 24小时以内 |

### 年度热点日历

**Q1（1~3月）**：
- 元旦/春节（1月）
- 情人节（2月）
- 妇女节/消费者权益日（3月）

**Q2（4~6月）**：
- 清明节（4月）
- 五一劳动节（4~5月）
- 母亲节（5月）
- 618购物节（6月）

**Q3（7~9月）**：
- 暑假热点（7~8月）
- 七夕（8月）
- 教师节（9月）
- 中秋节（9月）

**Q4（10~12月）**：
- 国庆节（10月）
- 双11（11月）
- 双12（12月）
- 圣诞节/元旦（12月）

### 热点内容制作优先级

| 热点类型 | 响应速度 | 内容深度 | 制作成本 |
|----------|----------|----------|----------|
| 突发热点（如明星事件） | <2小时 | 浅 | 低 |
| 可预见热点（节假日） | 提前1~4周 | 深 | 中 |
| 行业热点 | 24~48小时 | 中 | 中 |
| 平台热点 | 即时 | 浅 | 低 |
