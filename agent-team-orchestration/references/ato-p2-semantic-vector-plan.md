# P2 语义向量优化方案

**版本**: v1.0  
**创建时间**: 2026-05-24  
**优先级**: P2

---

## 1. 背景与动机

### 1.1 当前局限
TF-IDF 基于词频统计，无法理解语义：
- "如何设计多智能体系统" vs "多智能体架构最佳实践" —— TF-IDF 视为不同查询
- 无法处理语义相似但用词不同的情况

### 1.2 语义向量优势
- 捕获语义相似性，不限于关键词匹配
- 支持模糊查询和自然语言描述
- 可处理跨语言语义（如中英文混合查询）

---

## 2. 架构设计

```
用户查询 → 查询编码器 → 查询向量
                              ↓
专家知识文件 → 文档编码器 → 文档向量库 → 相似度计算 → Top-K推荐
```

---

## 3. 嵌入模型选择

**推荐**: text2vec-base-chinese
- 维度: 768
- 中文支持: 优秀
- 许可: Apache 2.0
- 本地部署，无需API

---

## 4. 向量存储方案

**推荐**: FAISS (CPU版本)
- 支持大规模向量
- 多种索引类型
- 速度快，无额外服务依赖

---

## 5. 实施步骤

### Step 1: 环境准备（1天）
安装 sentence-transformers 和 faiss-cpu

### Step 2: 向量编码模块（2天）
创建 semantic_encoder.py，封装编码和索引构建

### Step 3: 文档向量化（2天）
将所有专家文件编码为向量，构建 FAISS 索引

### Step 4: 混合检索（2天）
融合 TF-IDF 和语义向量结果，可配置权重

### Step 5: 集成优化（2天）
更新 team-brain.py，支持三种检索模式

---

## 6. 验证标准

- 语义检索正确识别相似查询
- 单查询编码 < 50ms，检索 < 20ms
- 整体准确率 >= 90%
- 内存占用 < 500MB
