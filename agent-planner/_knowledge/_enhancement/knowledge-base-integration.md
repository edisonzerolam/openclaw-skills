# knowledge-base 检索路径（Tier-1 内联）

> 版本：v1.0 | 状态：active
> 来源：agent-planner v2.9 → Tier-1 架构方案 增强
> 定位：当规划涉及特定领域时，检索本地知识库，融合行业知识提升规划质量

---

## 核心定位

**知识库优先原则**：规划时优先检索本地知识库，融合行业知识提升规划质量。

---

## 知识库目录结构

```
knowledge-base/                      # 相对于 workspace 根目录
├── medical-device/                  # 医疗器械
│   ├── README.md                   # 目录说明（必选）
│   └── *.md                        # 知识文档
├── bidding/                        # 投标标书
├── stock/                          # 股票
├── fund/                           # 基金
└── finance/                        # 理财
```

---

## 领域识别规则

| 触发关键词 | 知识库目录 | 优先级 |
|-----------|-----------|--------|
| 医疗器械、医疗设备、耗材、IVD | medical-device | P0 |
| 投标、标书、中标 | bidding | P0 |
| 股票、A股、港股、股价 | stock | P1 |
| 基金、净值、ETF、公募 | fund | P1 |
| 理财、资产配置、投资组合 | finance | P1 |

---

## 检索流程

```
用户输入规划请求
       ↓
   领域识别（关键词匹配）
       ↓
   knowledge-base/{domain}/ 读取所有 .md/.txt
       ↓
   语义匹配（内置）
       ↓
   匹配结果（前3个最相关片段，≤500字）
       ↓
   融入规划上下文
       ↓
   生成专业化规划方案
```

---

## 降级策略

| 场景 | 降级方案 | 自动/手动 |
|------|---------|----------|
| 知识库目录不存在 | 跳过检索，直接规划 | 自动 |
| 检索结果为空 | 使用通用规划能力 | 自动 |
| 领域无法识别 | 使用通用规划能力 | 自动 |
| 检索结果超500字 | 截断至500字，标注"已截断" | 自动 |

---

## 路径说明

- **知识库路径**：`knowledge-base/`（相对于 workspace 根目录）
- **示例**：workspace = `c:/Users/ediso/.openclaw/workspace`，则知识库完整路径为 `c:/Users/ediso/.openclaw/workspace/knowledge-base/`
