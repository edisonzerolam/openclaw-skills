# 知识工程规划模式（Knowledge Engineering Pattern）

> 本模式将39个知识文件、18项专家增强的规划过程固化为可复用的知识工程流程。
> 适用于 agent-planner skill 的 F1~F12 扩展场景。

---

## 1. 知识工程任务识别（F1扩展）

### 1.1 触发关键词识别

当用户提及以下任一关键词时，自动识别为「知识工程任务」：

| 触发词 | 任务类型 | 说明 |
|--------|----------|------|
| 蒸馏 | 单/多文件增强 | 从专家理论提炼为知识文件 |
| 创建知识 | 新团队模板初始化 | 全新知识文件体系搭建 |
| 增强知识库 | 多文件并行增强 | 对现有知识文件补充专家理论 |
| 专家理论注入 | 单文件定向增强 | 向指定文件注入特定专家内容 |

### 1.2 本次任务类型判定

**本次任务类型：多文件并行增强（P0/P1/P2三波批次）**

- 总文件数：39个知识文件
- 专家增强数：18项
- 批次规划：P0(5文件) → P1(6文件) → P2(4文件)
- 子代理调用：每文件1次，共15次

### 1.3 任务规模分级

```
S级：>20文件 或 >10项专家增强 → 建议分3波以上批次
A级：10~20文件 或 5~10项专家增强 → 建议分2波批次
B级：<10文件 或 <5项专家增强 → 可单波执行
```

## 2. 知识工程规划步骤（F2-F12扩展）

### F2 用户确认

在启动知识工程之前，必须与用户确认以下内容：

**确认清单：**
1. 增强范围：列出所有待增强文件（从知识库/专家领域匹配）
2. 优先级确认：P0/P1/P2 分类是否合理（高价值文件优先P0）
3. 来源要求：是否需要真实专家来源标注（本skills严格要求来源真实性）
4. 质量要求：是否需要量化阈值/公式/表格（如不需要可简化）

**本次确认内容：**
```
增强范围：15个知识文件（含股票分析/宏观预测/港股分析/替代数据/内容运营等）
优先级：P0=5文件(护城河/产能周期/做空狙击/因子评估/抖音流量池)
        P1=6文件(企业合规/平台适配/小红书运营/宏观信用等)
        P2=4文件(补充增强)
来源要求：必须标注真实专家姓名+机构，禁止虚构
```

1. 步骤一

### 交接协议

- 后续知识文件：xxx.md

### 质量检查点

- [ ] 检查项2

### 专家增强章节（可选）

### F7 坑点预防

#### 7.1 调研类任务超时预防

**坑点现象：** P0调研任务全部超时（5分钟不够）

**根因分析：** 调研+写入双任务耗时长，子代理5分钟限制不足

**解决策略：**
- 调研类任务设置2分钟超时
- 主会话保留备选执行能力
- 批次规模控制在≤3文件/批

**教训：** 并行批次中超过3个文件 → 必有超时，控制在≤3个/批

#### 7.2 插入位置错误预防

**坑点现象：** 原计划插入位置与实际不符（后续章节编号冲突）

**解决策略：**
- 写前：grep确认目标章节是否存在及行号
- 写时：新插入章节用「在现有第X节之后新增」而非「在末尾新增」
- 写后：grep章节标题 + 统计行数增长

#### 7.3 专家来源虚构预防

**坑点现象：** 子代理可能虚构专家来源

**解决策略：**
- 每个专家增强必须标注「真实姓名+真实机构」
- 禁止使用「某分析师」「业内人士」等模糊来源
- 如来源不确定，标注「待验证」而非虚构

## 3. 知识工程的成功标准

### 3.1 量化成功指标

| 指标 | 标准值 | 测量方法 |
|------|--------|----------|
| 行数增长 | >50行/文件 | `wc -l` 前后对比 |
| 公式数量 | ≥1个/增强章节 | grep公式符号 `$` |
| 表格数量 | ≥1个/增强章节（可选） | grep `\|` 分隔符 |
| 来源标注 | 100%覆盖 | grep `*{`*`}*` 格式 |

### 3.2 质量成功指标

- 每个增强章节含≥1个公式或≥1个量化表格
- 专家来源标注格式：`*{来源专家（机构）}*{主题}*`
- 无虚构专家来源
- LaTeX公式内无中文字符

### 3.3 来源标注格式规范

```
正确格式：
  *{邱国鹭（高毅资产）}*{护城河五分类量化标准}*
  *{任泽平（恒大研究所）}*{产能周期三因子公式}*

错误格式：
  *{某知名基金经理}*{护城河理论}*  ← 禁止模糊来源
  *{高毅资产邱国鹭}*{价值投资}*    ← 机构名应加括号
```
