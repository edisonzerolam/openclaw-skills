# agent-planner 审计协议（阶段A）

## 审计目标

| 维度 | 内容 | 通过标准 |
|------|------|---------|
| 完整性 | 规划是否覆盖所有必要项？Tier-1/Tier-2 是否齐全？ | 质量维度全部≥3/5 |
| 准确性 | 事实/数据/引用是否准确？ | 无虚假/夸大描述 |
| 一致性 | 规划与调查摘要/rules.md 是否一致？ | 无矛盾 |
| 可执行性 | Task 是否可直接执行？依赖/步骤是否清晰？ | 示例可复现 |
| 风险控制 | 坑点≥3 且降级策略完整？ | 风险预警充分 |

## 执行流程

```
规划方案产出
  → 执行自检（质量维度评分）
  → 调用事实核查工具
  → 审计评分
  → 审计通过/不通过
    → 通过：提交 F2 用户确认
    → 不通过：返回 F6 修订，上限 5 版
```

## 检查清单

| 检查项 | 标准 | 不通过时 |
|--------|------|---------|
| Tier-1 全部 5 项已产出 | 问题诊断/架构方案/技能选型/工作流/坑点预警 | 阻塞 |
| 坑点预警≥3 个 | F7 匹配相关坑点 | 阻塞 |
| 数字准确性 ⚠️ | 规划中数字声明经 F0.5 验证通过 | FAIL→阻塞，WARN→标记 |
| 架构方案含降级策略 | L1-L4 降级路径完整 | 警告 |
| 工作流含异常处理 | 每个 Step 有降级处理 | 警告 |
| 环境配置含验证命令 | 依赖验证命令存在且正确 | 警告 |
| Task 示例可直接执行 | 示例在 workspace 可复现 | 阻塞 |
| 成本预估符合预算 | Token/调用次数在约束内 | 警告 |
| 模型调用效率≥3/5 | API 调用链路最短/去重/退避 | 警告 |

```markdown

- 审计时间：{YYYY-MM-DD HH:mm}
### 各项评分
| 维度 | 评分 | 说明 |
|------|------|------|
| 完整性 | X/5 | {说明} |
| 可执行性 | X/5 | {说明} |
| 风险控制 | X/5 | {说明} |
| Token 效率 | X/5 | {说明} |
| 模型调用效率 | X/5 | {说明} |
### 阻塞项
- {问题} → {修复建议}
### 警告项
- {问题} → {建议}
### 综合评分

```

## 通过条件

- 综合评分 ≥3.5/5
- 无阻塞项（或已修复）
- 迭代上限 5 版，超限强制用户决策

## 质量自检

| 维度 | 评分 | 说明 |
|------|:----:|------|
| 完整性 | X/5 | Tier-1 全部 5 项已产出 |
| 可执行性 | X/5 | 任务示例可直接执行 |
| 风险控制 | X/5 | 坑点≥3 且降级策略完整 |
| Token 效率 | X/5 | Context 精简/batch 缓存 |
| 模型调用效率 | X/5 | API 调用最短链路/去重 |

综合≥3.5 → 通过；<3.0 → 需改进

## 版本迭代控制

- 迭代上限：5 版，超 5 版强制用户决策门
- 版本号格式：`v{主}.{次}`
- 每版记录变更摘要到 `_knowledge/_enhancement/plan-tracker/evolution-log.md`
