## A8. 质量门 Q7 — 数字现场核查（2026-05-22 新增）

> **来源**：本增强层补充，编号 Q7（优先于 Q1）
> **定位**：Layer0 质量门，Q0 之后 Q1 之前执行
> **触发条件**：子代理引用报告中的数字作为决策依据时

### A8.1 Q7 触发判定

```
当以下任一场景出现时，必须执行 Q7 数字现场核查：

[ ] 引用报告中的文件行数、大小、路径值
[ ] Expert Panel 类任务依赖历史报告数据
[ ] 涉及代码行数、函数签名、配置值的变更评估
[ ] 任何"SHA256一致"声明

Q7 执行时机：Phase-S S1 战略评估阶段（S4 验证前）
```

### A8.2 Q7 核查标准

| 数字类型 | 核查命令（PowerShell） | 偏差阈值 |
|---------|---------------------|---------|
| 文件行数 | `(Get-Content path \| Measure-Object -Line).Lines` | ±10% → warn |
| 文件大小 | `(Get-Item path).Length / 1KB` | ±10% → warn |
| 目录总大小 | `(Get-ChildItem dir -Recurse -File \| Measure-Object -Property Length -Sum).Sum / 1KB` | ±10% → warn |
| 路径值 | `Get-Content file \| Select-String "pattern"` | 性质不符 → block |
| 函数/方法行数 | `Select-String -Path file -Pattern "^    def name"` | ±10% → warn |
| SHA256一致性 | `Get-FileHash core\file | Select-Object -ExpandProperty Hash` 对比 | 不一致 → warn |

### A8.3 Q7 判定规则

```json
{
  "gate": "Q7",
  "status": "pass|warn|block",
  "deviations": [
    {
      "claimed": "575行",
      "actual": "504行",
      "file": "backtest_engine.py",
      "deviation_pct": 14,
      "action": "warn"
    }
  ]
}
```

### A8.4 Q7 与 R6 的关系

- Q7 是**执行层**（核查动作）
- R6 是**决策层**（判断是否允许引用未经核查的数字）
- Q7 pass + R6 warn：数字已核查但应谨慎引用
- Q7 fail + R6 block：数字偏差过大或性质不符，停止引用

> **触发词**：「专家小组」「评估报告」「多人审计」「second opinion」

### A9.1 M 层执行流程

```
