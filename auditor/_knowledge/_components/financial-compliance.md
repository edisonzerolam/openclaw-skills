# 增强层L — 财务合规审计域

> 版本：v1.0 | 状态：active
> 来源：auditor v1.0.0 财务合规审计 + v6.1 融合框架
> 定位：当 G1 = `财务合规审计` 时，内化为增强层L，注入 S1/S3/S5

---

## 核心定位

财务合规审计作为独立域，无缝嵌入 auditor v6.1 的 Phase-S 执行框架。当用户请求财务审查、内部控制评估、合规性检查时，触发此增强层。

---

## G1 = 财务合规审计 的路由

```
Phase-G G1 分类 = "财务合规审计"
    ↓
增强层 L 激活
    ↓
S1 → S1+L（财务域战略评估）
    ↓
S3 → S3+L（财务域 Merge Gate）
    ↓
S5 → S5+L（财务域结果沉淀）
```

---

## S1+L — 财务域战略评估

### L1 定义审计范围

| 类型 | 说明 |
|------|------|
| 财务审计 | 财务记录准确性、账目核对、报表审计 |
| 运营审计 | 业务流程效率、合规性 |
| IT审计 | 信息系统控制、数据安全 |
| 内部控制审计 | 控制环境、控制活动、监控 |

### L2 确立审计目标

| 目标 | 说明 |
|------|------|
| 准确性 | 财务记录是否真实、完整、准确 |
| 合规性 | 是否符合 GAAP/IFRS/内部政策 |
| 风险 | 识别财务风险和控制弱点 |
| 效率 | 流程是否优化，是否存在冗余 |

### L3 设定重要性阈值（Materiality）

```yaml
materiality_threshold:
  整体: {金额或百分比，如收入的1%}
 组成部分: {各科目重要性阈值}
 实质性门槛: {需立即上报的金额}
```

### L4 识别适用标准

| 标准 | 适用场景 |
|------|---------|
| GAAP | 美国通用会计准则 |
| IFRS | 国际财务报告准则 |
| 企业内部政策 | 内部控制的最佳实践 |
| 监管要求 | 行业特定监管规定 |

### L5 关联增强层J（知识库）

加载领域知识：
- 财务报告标准
- 内部控制框架（如 COSO）
- 行业特定合规要求

---

## S3+L — 财务域 Merge Gate

### 额外检查项

| 检查 | 说明 |
|------|------|
| 审计证据链完整性 | 是否有足够的支撑文件 |
| 数字一致性 | 账目、表单、数字是否勾稽一致 |
| 重大错报风险 | 是否存在重大错报可能性 |
| 控制弱点识别 | 关键控制点是否有效 |

### 风险评级映射

| auditor L1-L4 | 财务审计风险 |
|---------------|-------------|
| L4 | High（高风险） |
| L3 | Medium-High |
| L2 | Medium |
| L1 | Low（低风险） |

> 财务域使用 High/Medium/Low 三级，不同于系统优化的 L1-L4。

---

## S5+L — 财务域结果沉淀

### 发现分类

| 类别 | 说明 |
|------|------|
| 财务准确性 | 账目错误、报表问题 |
| 合规性违规 | GAAP/IFRS/政策违规 |
| 控制缺陷 | 内部控制弱点 |
| 运营低效 | 流程优化机会 |

### 风险评级（High/Medium/Low）

| 风险 | 定义 | 行动 |
|------|------|------|
| High | 可能导致重大财务损失或合规违规 | 立即修复，P0 |
| Medium | 可能导致中等损失 | 近期修复，P1 |
| Low | 不影响整体财务健康 | 计划改进，P2 |

### 纠正措施模板

```yaml
finding:
  id: FC-{seq}
  category: {财务准确性/合规性违规/控制缺陷/运营低效}
  description: "{问题描述}"
  risk_level: {High/Medium/Low}
  evidence: ["{证据1}", "{证据2}"]
  recommendation:
    priority: {P0/P1/P2}
    action: "{具体纠正措施}"
    owner: "{负责人或部门}"
    deadline: "{修复期限}"
```

---

## 与系统优化审计的协同

| 场景 | 处理方式 |
|------|---------|
| 纯财务合规审计 | 仅启用增强层L，跳过系统优化增强层A-K |
| 混合审计（系统+财务） | L与A-K同时生效，S1/S3/S5分别注入 |
| 系统变更触发财务相关 | 识别关联，自动启用L中的财务影响评估 |
