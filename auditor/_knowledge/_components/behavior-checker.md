# Q6 行为红线规则引擎

> 版本：v1.1 | 状态：active（内置）
> 来源：auditor Layer0 Q6 强制检查
> 定位：审计行为红线 R1-R6，对应 rules.md 的核心原则
> 更新：2026-05-22 新增 R6 规则（不凭二手数字决策）

---

## 核心定位

auditor v6.2 强制执行 6 条行为红线（对应 rules.md），确保审计过程不越界、不失控。

---

## 规则定义

### R1 不加没要求的功能

**判断**：Feature 是否超出原始需求范围
**锚点**：Feature ≠ 原始需求
**严重性**：P0
**动作**：block

### R2 不过度封装

**判断**：是否存在同类形态可复用
**锚点**：同类形态已存在
**严重性**：P1
**动作**：warn

### R3 不瞎重构

**判断**：是否有 E2E 测试证明 + 数字现场核查（双重门）
**锚点**：无 E2E 测试 **且** 无数字现场核查记录
**严重性**：P0
**动作**：block
**补充**：有 E2E 测试但无现场数字核查 → warn（R6 触发）

### R4 不假装测试通过

**判断**：是否有实际执行记录
**锚点**：无执行记录
**严重性**：P0
**动作**：block

### R5 先读再改不准盲改

**判断**：修改前是否已读取原文件
**锚点**：修改前未读取
**严重性**：P1
**动作**：warn

### R6 不凭二手数字决策（2026-05-22 新增）

**判断**：关键数字（行数/文件大小/路径值/配置值）是否经过现场 exec 验证
**锚点**：引用报告中的数字，但未执行 exec 现场核查即作为决策依据
**严重性**：P1
**动作**：warn

**触发场景**：
- 子代理引用报告中的数字作为结论依据
- Expert Panel 类任务依赖历史报告数据
- 涉及代码行数、目录大小、函数签名的变更评估

**Q7 数字核查标准**（对应 auditor 新增质量门 Q7）：

| 数字类型 | 核查方式 | 偏差阈值 |
|---------|---------|---------|
| 文件行数 | `Get-Content file \| Measure-Object -Line` | ±10% → warn |
| 文件大小 | `(Get-Item file).Length / 1KB` | ±10% → warn |
| 路径值 | `Get-Content file \| Select-String pattern` | 性质不符 → block |
| 函数行数 | `Select-String "^    def name"` | ±10% → warn |

**R6 与 R3 的关系**：R3 要求 E2E 测试；R6 要求数字现场核查。两者共同构成"不盲改"双重门。

---

## 执行方式

Q6 在 Layer0 执行，遍历 R1-R6 逐项检查。

- P0 违规 → block，停止审计，要求用户确认
- P1 违规 → warn，记录到 audit-template.md，继续审计
