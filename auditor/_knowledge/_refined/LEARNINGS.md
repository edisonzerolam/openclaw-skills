# auditor 自学习经验沉淀

> auditor 自学习机制的经验积累与知识沉淀
> 版本：v1.1 | 2026-06-03（编码修复+内容重建）
> 触发条件：同一类变更>=3 次出现缺陷记录

---

## 概述

LEARNINGS 是 auditor 的自学习知识库，记录从实际执行中提炼的经验、教训和改进建议。
当 S5.9 进化引擎触发时，将从 LEARNINGS 中提取相关经验用于技能优化。

---

## 触发机制

### S5.9 进化引擎流程

```
触发条件：同一类变更>=3 次出现缺陷记录
    |

    |
```

| 场景 | 触发条件 | 学习来源 |
|------|---------|---------|
| 变更缺陷 | 同一类变更>=3 次出现缺陷 | 缺陷记录 → 教训提炼 |

## 沉淀流程

### 步骤 1：记录（Recording）

每次缺陷/问题发生时，在对应的 daily memory 中记录：

```markdown
### YYYY-MM-DD HH:MM
- **类型**：变更缺陷/质量门失败/超时/知识缺失
- **现象**：具体描述
- **根因**：根本原因分析
- **教训**：提炼的经验教训
- **建议**：改进建议
```

### 步骤 2：归类（Categorization）

将记录归类到对应的经验类别：

- 变更分类经验 → G1 相关
- 质量门经验 → Layer0 相关
- 执行框架经验 → S1-S5 相关
- 知识工程经验 → 知识库相关

### 步骤 3：泛化（Generalization）

从具体案例提炼通用规律：

```
具体案例 → 抽象模式 → 通用规则
```

### 步骤 4：沉淀（Deposition）

将通用规则写入 LEARNINGS.md 对应章节。

### 步骤 5：应用（Application）

更新 SKILL.md 或相关组件，使经验可被后续执行利用。

## 经验条目

### [多源报告数字偏差 — 专家小组场景]

**沉淀日期**：2026-05-22
**触发次数**：1次（首次触发，作为典型案例记录）
**触发条件**：专家小组任务依赖历史报告数据，未执行独立实地核查

**现象**：
- backtest_engine.py 行数：报告575行，实际504行（+14%偏差）
- strategies/ 冗余量：报告93KB/7/8 SHA256一致，实际46.7KB/4文件真镜像（+98%偏差）
- selector.py 路径：报告第38行绝对路径硬编码，实际第37行是`__file__`相对路径（误报）

**根因**：
- 子代理直接接受历史报告数字为事实，未执行 exec 验证
- Phase-S 框架无 Q7 数字现场核查门
- R1-R5 无"不凭二手数字决策"规则
- 原报告"7/8 SHA256"为捏造数字（实际仅4个真镜像）

**教训**：
- 历史报告中的数字是"快照"，文件可能被修改
- 任何关键数字（行数/大小/路径）在使用前必须现场验证
- 专家小组结论 ≠ 实地核查结论
- "SHA256完全一致"需用实际命令验证，不能凭文件名相同推测

**正确做法**：
1. 子代理执行任务时，对关键数字执行 Q7 强制核查
2. 偏差>10%的数字以实地为准，报告数字标记为"待核实"
3. Q7 应在 Phase-S S1 阶段执行，而非 S4 验证阶段
4. 数字声称（SHA256/行数/大小）必须有 exec 执行记录

**受影响文件**：
- auditor/_knowledge/_components/behavior-checker.md（已追加 R6）
- auditor/SKILL.md（待更新 Layer0 + Q7）

**相关SKILL**：agent-planner（P-Sub-P 规划修正可参考此经验）

### [agent-planner F0.5 事实验证机制 — 协作反馈]

**沉淀日期**：2026-06-03
**触发次数**：1次（基于多源报告偏差案例的系统性修复）
**触发条件**：auditor 发现 agent-planner 规划产物数字不准确

**经验内容**：
- 规划产物中的数字声明必须经 F0.5 事实验证门
- 使用 fact_check.verify_source_claims() 执行现场验证
- FAIL（偏差>30%）阻塞 F1，WARN（10%-30%）用户决策

**正确做法**：
1. agent-planner F0 调查摘要产出后，执行 F0.5 验证
2. 验证结果写入调查摘要 verification_results 字段
3. 审计阶段检查"数字准确性"

**受影响文件**：
- agent-planner/SKILL.md（已新增 F0.5 子阶段）
- agent-planner/frozen_version.json（已更新至 v3.4）
