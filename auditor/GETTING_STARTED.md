# Auditor 快速入门

> 版本：v1.0 | 5 分钟上手 auditor v6.2

---

## 案例 1：Skill 变更审计

**场景**: 修改了一个 Skill 的 SKILL.md

**步骤**:
1. 触发词："审计这个 skill 变更"
2. Phase-G: 系统自动分类为 "Skill变更"
3. Layer0: Q0-Q6 质量门自动检查
4. S1: 评估风险等级（通常 L2）
5. S4: 8 项验证（文件存在/数值一致/路径正确等）
6. S5: 输出审计报告

**预期输出**: `auditor-{skill-name}-audit-{date}.md`

---

## 案例 2：财务合规审计

**场景**: 审查财务报表准确性

**步骤**:
1. 触发词："财务审查" / "内部控制检查"
2. 增强层 L 自动激活
3. S1+L: 定义审计范围 + 重要性阈值
4. S3+L: 审计证据链完整性检查
5. S5+L: 按 High/Medium/Low 评级输出发现

**预期输出**: 含 `financial_audit_section` 的审计报告

---

## 案例 3：系统变更审计

**场景**: 修改 OpenClaw Gateway 配置

**步骤**:
1. 触发词："系统诊断" / "变更审查"
2. Phase-G: 分类为 "系统变更"
3. S1: 风险等级 L3/L4（系统变更高风险）
4. S2: 规划子代理（可能并行）
5. S4: 强制 8 项验证 + E5 备份
6. S5: 结果沉淀 + 进化引擎

**预期输出**: 含性能优化建议的审计报告

## 常用触发词

| 触发词 | 分类 | 增强层 |
|--------|------|--------|
| "审计" | 自动判断 | 按需 |
| "优化建议" | Skill/系统 | K |
| "变更审查" | 变更实况 | A |
| "系统诊断" | 系统变更 | C+D |
| "系统健壮" | 系统变更 | C+D |
| "财务审查" | 财务合规 | L |
| "内部控制" | 财务合规 | L |
| "合规检查" | 财务合规 | L |

## 质量门速查

| 检查 | 通过标准 | 失败处理 |
|------|---------|---------|
| Q0 Context | 健康 | 建议压缩 |
| Q1 依赖 | 无关键缺失 | 降级增强层 |
| Q2 确认 | 已授权 | 请求确认 |
| Q3 复杂度 | ≤5 变更 | 触发 Phase-D |
| Q4 冻结 | 未冻结 | 禁止变更 |
| Q5 Workspace | 路径正确 | 修正路径 |
| Q6 红线 | 无 P0 违规 | Block |

---

## 风险等级

| 等级 | 说明 | S4 要求 |
|------|------|---------|
| L1 | 低风险 | 可选跳过 |
| L2 | 中风险 | 建议执行 |
| L3 | 高风险 | 强制执行 |
| L4 | 极高风险 | 强制 + E5 备份 |

## 增强层速查

| 层 | 触发条件 | 用途 |
|----|---------|------|
| A | CI/CD 变更 | 历史审计 |
| B | 所有审计 | 行为红线 |
| C | 所有审计 | Context 健康 |
| D | 子代理 | 会话管理 |
| E | 复杂规划 | 规划修正 |
| F | 需要调研 | 深度研究 |
| G | S5 结尾 | 进化引擎 |
| H | 子代理执行 | 团队协调 |
| I | 报告输出 | 文档生成 |
| J | 知识需求 | 知识库 |
| K | 质量检查 | 5 维质量 |
| L | 财务触发词 | 财务合规 |
