# auditor 审计协议（阶段A）

## 审计目标

| 维度 | 内容 | 通过标准 |
|------|------|---------|
| 完整性 | Phase-G + S1-S5 是否齐全？ | 质量维度全部≥3/5 |
| 准确性 | 事实/数据/引用是否准确？ | 无虚假/夸大描述 |
| 一致性 | 与调查摘要/rules.md 是否一致？ | 无矛盾 |
| 可执行性 | Task 依赖/步骤是否清晰？ | 示例可复现 |
| 风险控制 | 风险评级合理 + 降级策略完整？ | 风险预警充分 |

## 执行流程

```
审计方案产出 → 自检（质量评分）→ 事实核查 → 审计评分
  → 通过：提交 Phase-G 确认
  → 不通过：返回修订（上限 5 版）
```

## 检查清单

| 检查项 | 不通过时 |
|--------|---------|
| Phase-G 全部 5 项已产出 | 阻塞 |
| S1 包含风险等级 | 阻塞 |
| S3 Merge Gate 通过 | 阻塞 |
| S4 验证覆盖全部 8 项 | 警告 |
| S5 经验归档完整 | 警告 |
| 增强层 A-L 使用合理 | 警告 |
| 财务合规（如适用）：High/Medium/Low 映射正确 | 阻塞 |

## 审计结果格式

```markdown
## 审计报告
### 审计结果：通过 / 不通过
- 审计版本：v{版本号}
- 审计时间：{YYYY-MM-DD HH:mm}
### 各项评分
| 维度 | 评分 | 说明 |
|------|:----:|------|
| 完整性 | X/5 | {说明} |
| 准确性 | X/5 | {说明} |
| 一致性 | X/5 | {说明} |
| 可执行性 | X/5 | {说明} |
| 风险控制 | X/5 | {说明} |
### 阻塞项 / 警告项
- {问题} → {修复建议}
### 综合评分
**综合评分**：X.X/5 → {通过/需改进/不合格}
```

## 通过条件

- 综合评分 ≥3.5/5
- 无阻塞项（或已修复）
- 迭代上限 5 版，超限强制用户决策
